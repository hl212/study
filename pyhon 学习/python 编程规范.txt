#coding=utf-8

1、缩进 —— 使用空格而不是制表符
2、空行
     用两行空行分割顶层函数和类的定义,类内方法的定义用单个空行分割.
3、编码
     建议 统一UTF-8


说明:命名约定
    Python包应该是不含下划线的,简短的,全小写的名字
    类名总是使用首字母大写单词串
    函数名应该为小写,可能用下划线风格单词以增加可读性.
    方法名和实例变量   通常使用小写单词,必要时用下划线分隔增加可读性 ，使用一个前导下划线仅用于不打算作为类的公共接口的内部方法.  使用两个前导下划线以表示类私有的名字.


文档：doc
    __init__.py 中的注释成为包文档
    文件头部注释成为模块文档
    紧贴 class 声明后的注释成为类文档
    紧贴 def 声明后的注释成为函式文档


建议：
    尽量避免 from module import * （有时找出一个命名空间是一个很痛苦的事情）
